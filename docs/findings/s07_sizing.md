## 7. サイジング/期間成長（「増やす」文脈）
- 🔑 **「ギャンブルめに増やす」＝1回で大きいRでなく、賭け率 f を上げて"期間"複利**。評価は期間倍率の分布（中央値・標準偏差・破産/半減確率）。
- 🔬 **中央値2倍(≥50%) ⇔ N·E[ln(1+f·r)] ≥ ln2**（分散非依存）。レバー＝**回数N・PF・f の3点**。例(RR1): PF2.0で倍化に 年107本@2%/45本@5%/24本@10%。`double_in_year.py`
- 静的inv-volが最適（cross-leg momentum無し=reversal）。vol-targeting/動的アロケはOOSをISの劣化で買う偽=却下。
- ❌🔑 **同一レッグの同時複数ポジション（max-pos≥2）＝「劣化した f 上げ」、棄却**（2026-07-03, `breakout_wave.py --max-pos` 実装・P=1はbit一致）。BTC15m/4hゲート/frac0.3: P=2でN/yr 102→169・totR/yr 36.5→44.2(+21%)だが、**増分496本の質はmeanR+0.097/PF1.11**（同じトレンドの複製）で、相関リスクの積み上がりにより**複利軸で全敗**——f2%: P(半減)0.4→3.8%、f3%: 1.9→**10.8%**・中央値2.26→1.94。「P=1のままfを上げる」が常に支配。P=3以上は希薄化でtotRも崩壊（P=∞でmeanR+0.07）。**∴busyに潰されるシグナルは同一の動きの重ね取りでしかない。Nは独立な機会（銘柄・入口族）からのみ増やせる＝頻度法則の別表現。**

- ❌🔑 **「上位足のトレンドが強い時に大きく張る」＝レッグは大幅改善するがブックを壊す（構造法則6の実例、採用せず）**（`scratchpad/btc15m_htf_size.py` `btc15m_v2_audit.py` `book_v2_judgement.py`, 2026-07-13）。機構の出発点=「入口の強さは目標でなくサイズの変数」（s02）。**変数**: HH4H＝ブレイク終値が**直近の確定4Hスイング高値**を上抜けているか（先読み対策: 4Hスイングは確定足の終値で初めて既知なので **4H足で1本shiftしてから15分足に展開**。これを怠ると最大4時間の先読みになる）。PDHとの冗長性は無い（該当率46% vs 49%、両方満たすのは31%、P(PDH上|4H高値上)=68%）。**レッグ単体では今日最強**: サイズ階段（両方=1.0 / どちらか=0.5 / どちらも無し=0.25）で **PF 1.62→1.75・maxDD 20.4%→10.9%・CAGR/DD 1.99→3.02・2022 −13R→−5.6R**（トレード集合は同一n=763＝見送らず玉数だけ変える）。**ブロック・ブートストラップも通過**: P(勝ち)=78%(1か月)/88%(3か月)/96%(6か月)/**97%(12か月)** ＝**ブロックを長くするほど強くなる＝経路依存でない本物の署名**（週足ERゲートの逆）。**「弱い時は見送る」（ゲート化, 弱ウェイト0）は7〜9%で惨敗＝見送らず小さく張るのが正解**。定義の頑健性も合格（ZigZag k=1.5/2.0/2.5/3.0 で PF 1.72〜1.77、**ZigZagを捨てて「直近N本の4H高値」に替えても PF 1.70〜1.75**＝検出器の癖でなく概念が効いている）。監査も合格（DSR@240=1.00・PBO 0.24（240構成のCSCV、ノイズ対照0.64）・CAGR/DD CI[1.17, 2.52, 4.68]・null p=0.0000）。**しかしブック審判で却下**: 6レッグ・inv-vol・総リスク3%で **ブックCAGR/DD 8.55 → 6.96**（当時の月次DD審判。**トレード解像度の再審でも 6.84 → 6.20 で却下は維持**）。**理由は相関**: btc15m_L と btc_bo_kama の月次相関 +0.39→+0.42、gold_bo +0.10→+0.22 ＝ **小さく張るようにした「弱いセットアップ」こそがブックの他レッグと無相関だった玉**。削ると「上位足が強い時だけ大きく張る純粋なトレンド機械」になり、それは btc_bo_kama / gold_bo が既にやっていること＝分散が消える。レッグのDDは半減したのにブックのDDは 4.7%→5.6% に増加。∴**構造法則6（エッジと独立性はトレードオフ）の教科書的実例。レッグではなくブックで裁定する規律が仕事をした**。**残る使い道**: 単独運用（ブックに入れない）なら有力、または弱ウェイトを0.25でなく0.5〜0.75に緩めて分散を残す形は未検証。
- ❌🔑 **【撤回 2026-07-13】「BTCが直近4週間で走った直後のトレードは玉を減らす」＝ブックの月次DD審判が生んだ幻**（`scratchpad/book_healing_veto.py` `book_bootstrap_arbiter.py`, 2026-07-13, 母集団=採用形 btc15m_L（正典 breakout_wave.run・4hKAMAゲート・押し目指値0.3・RR4.5・PDHソフト0.5）、文脈は全て前日確定足）。**出自**: サイクル検証（s03）の副産物「最高値へ急回復中のトレードが死荷重」。**まず変数の正体を確定**: 「4週間のDD-from-ATH改善」と「単純な4週間リターン」の相関は **+0.820** ＝**サイクル変数ではなく伸びすぎ（over-extension）変数**。両者ほぼ同結果なので**単純な4週間リターンを採用**（gold15m の ext-cap・btc_pull の週足30MAゲートと同族）。**重要な訂正**: 「死荷重」は前の機械（日足ゲート・RR4・ラチェットON）での話で、**採用形の上では該当群は黒字**（meanR +0.314・PF 1.58・全体の20%）＝**削っているのは勝ちトレードであり、利得は純粋に分散から来る**（それらの月はブックの他レッグ＝全部トレンドフォロワーが稼ぐ月と重なる）。**数字**: 該当トレードの重みを 0.5 にすると単一経路のブック CAGR/DD **12.03 → 12.70**（レッグ単体は CAGR/DD 3.20→2.79 と**悪化する**＝HH4Hと真逆）。**ブロック・ブートストラップ（ブックの月次を再抽出）**: P(基準に勝つ) = **63%(1か月) → 70%(3か月) → 73%(6か月) → 78%(12か月)** ＝**ブロックを長くするほど強くなる＝本物の署名**。**台地も合格**: しきい値{65,70,75,80,85%点}×重み{0.25,0.5,0.75}の**15セル全部が 66〜85%**（最良=80%点×w0.75 で85%）。**全カット(w=0)より半減(w=0.5)が良い**＝「見送るな、小さく張れ」の再現。**留保**: 利得は控えめ（ブック +3〜10%）・15〜34%の確率で負ける・黒字トレードを削る形なので心理的に難しい・機構（該当月がブックの他レッグと高相関）の直接確認は未了。**【撤回の理由】ブック審判そのものが壊れていた**（反証チェックリスト8）。上の 12.03→12.70 は、ブックの maxDD を**月次**資産曲線で測っていたために生まれた数字で、その maxDD は **2019-07 の単月 −3.62%** ＝「最悪の1か月」そのものだった。btc15m_L はその月に16本しか建てておらず**合計 +17.5R の黒字**＝**ドローダウンの原因ですらない**。玉を絞ると btc15m_L の月次ボラが下がり inv-vol の重み配分が動き、たまたまその1か月の合成が薄まっていただけ。だから **21本削っても254本削っても答えが同じ**（13.37 / 13.36）という、機構なら絶対に出ない形が出た。**トレード解像度で maxDD を測り直すと 6.84 → 6.84 ＝完全な同値**、ブロック・ブートストラップも 63→70→73→78% が 63→63→58→**54%** に反転（ブロックを伸ばすほど50%へ縮む＝本物の署名の逆）。旧審判で最高だった q0.6/w0.0（14.24）は新審判で**基準割れ 6.76**。**構造法則8の例外候補は取り下げ**。検屍＝`scratchpad/ext_throttle_autopsy.py`、新審判＝`scratchpad/book_arbiter_v2.py`。なお Q2（巡回シフトnull）でも、btc15m_L 単独では多重比較後 p=0.129 で機構を立証できていない（ロング3レッグをプールしてやっと p=0.055）。
