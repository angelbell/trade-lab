- 🚨🔑🔬 **【ラボのバグ・要修正】ブックの maxDD を月次資産曲線で測ると、月内で完結するドローダウンが全部消える**（`scratchpad/book_arbiter_v2.py` `ext_throttle_autopsy.py` `book_dd_arbiter_check.py`, 2026-07-13）。
  **症状**: 6レッグ・ブック（inv-vol・総リスク3%）の maxDD が **3.62%**、CAGR 43.6% ＝ **Calmar 12**。6本の独立でないレッグを3%リスクで回して最大DDが3.6%というのは現実にはあり得ない。ピークからトラフまでを見ると **2019-06 → 2019-07 の「1か月」** ＝ **maxDD がブックの最悪の月そのもの**になっていた。
  **機構**: `book()` / `research/portfolio_alloc.py: cagr_dd_monthly()` は各レッグのRを**月次合計に潰してから**資産曲線を作る。月の中で「−3% まで沈んで +3% 戻して月末はゼロ」という経路は、月次では「0%」になり**ドローダウンとして存在しなくなる**。結果、CAGR/DD は「最悪の1か月をどれだけ薄められたか」だけを測る指標に化ける。
  **なぜ今まで牙を剥かなかったか**: 3レッグ時代のブックは月次DD **7.81%** と日次DD **7.80%** が一致していた（364トレード/8年＝月に数本しか建てないので、月内DDがほとんど発生しない）。**15分足レッグ（btc15m_L 758本・btc15m_S 99本）を足した瞬間に壊れた**。
  **影響**: トレード解像度で測り直すと A0 = CAGR 44.7% / maxDD **6.53%** / CAGR/DD **6.84**。**アームの順位が入れ替わる** — 月次審判で最高だった ext-throttle q0.6/w0.0（14.24）は新審判で**基準割れ 6.76**、「12.03→12.70」の改善は **6.84→6.84 の同値**に消えた。生き残ったのは **RR4.5**（新審判でもラダー最良 6.84 vs RR4.0 の 6.36）と **HH4Hサイズの却下**（6.20 / 5.65 < 6.84）の2件のみ。
  **新しい審判**: `scratchpad/book_arbiter_v2.py: trade_book()` — 重み（月次σの逆数・総リスク3%）は従来通り決めるが、**全レッグの全トレードを時刻順に1本に並べて `cumprod(1 + w_leg * R)` で複利し、その曲線から maxDD を取る**。ブロック・ブートストラップも月ブロックを並べ替えつつ**月内のトレード列は保つ**形に作り直した。
