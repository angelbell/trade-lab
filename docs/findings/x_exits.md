### 2.1 ユーザーのチャート観察に発した3案（2026-07-13、btc15m_L RR4.5・4Hゲート・6レッグ・ブック審判）
ユーザーが実チャート3枚から出した「損切りが刈られすぎ／構造が崩れたら降りろ」という観察を、そのまま機械化して反証した。
**3つとも死んだが、観察そのものは正しかった** — 死因はどれも「損切りを1Rに固定している以上、救える額は高々1R、切る額は最大4.5R」という非対称。

- ❌🔑 **損切りを L2（波2の安値）→ L0（波の起点）へ広げる＝ユーザーの知覚は正しいが、算術で負ける**（`experiments/btc15m_stop_repair.py` `stop_widen_autopsy.py`）。
  ユーザーの読み（「この損切りは刈られている」）は**当たっている**: 損切りを広げると**勝率 22.8% → 33.2%** に上がる。だが **totR +445 → +278**。
  **解剖1（恒等式）**: R は損切り幅を1とする単位。L0 にすると損切り幅の中央値が **$315 → $572（1.8倍）**になるが、
  **1勝あたりの獲得金額は $1,974 → $1,956 でほぼ不変** ＝ 同じ値幅を、2倍近い分母で割っているだけ。1勝の R は **5.90R → 3.43R** に半減する。
  **解剖2（同一エントリーでの直接対決, n=402）**: 現行で損切りされた **135本のうち、L0 なら勝ちに転じたのは 8本＝6% だけ**。

  ∴「損切りが浅くて刈られる」という**体感は、負けトレードの記憶が鮮明なだけ**で、母集団では成立しない。
- ❌ **R段階ラチェット（+xR 到達で損切を +yR へ）は RR4.5 でも死ぬ**（`experiments/btc15mL_stop_repair.py`, 16セル掃引）。

- ❌🔑 **構造退出（15分フラクタル・ピボットで「スパイク未超え → 高値切り下げ → サポート割れ」を判定して降りる）＝ランダム退出にすら負ける**（`experiments/btc15mL_pivot_exit.py`, 2026-07-13, 正典 tie-back 済み: n=759, max|dR|=3.4e-13）。

  **(1) 事前登録した反実仮想の分解で 40/40 セルが不合格**。全セルで **切ったR（cut_r）> 救ったR（saved_r）**。

### 2.2 「持ち続けられない相場」への対処 — 出口ではなくサイズで（2026-07-13, btc15m_A）
ユーザーの申告: **「統計上はそのほうがいいかもしれないが、戦争中で世界情勢的に持ち続けられない。1時間レベルのレジサポ転換で利確しようかと思う」**＝ 心理的制約であって、ギャップ・リスクの話ではない。
**守れないルールは価値ゼロ**なので、この申告は真剣に測る価値がある。だが結論は明快だった。

- ❌🔑 **「1時間足の直近の押し目安値を割ったら利確」＝ 同じ maxDD で CAGR が 10〜16ポイント落ちる**（`experiments/h1_swinglow_exit.py`。基準 = btc15m_A ＋日足×0.75、maxDD 8.48% に全腕をそろえて CAGR で比較）。
  1時間足の確定フラクタル安値（k=2/3/5本）を確定終値で割ったら翌足寄付で退出。発火条件も 4通り（常に / +0R以上 / +1R以上 / +2R以上）。**12セル全部が負け**: 常に発火 → CAGR **−16.5pt**、最も保守的な「+2R以上でのみ」でも **−10.7pt**。
  **反実仮想の分解（事前登録）**: どの設定でも **切った額が救った額の3倍以上**。k=3・常に発火: 救った 30本 +26.8R / **切った 20本 −73.9R** ＝ 差引 **−47.1R**。+2R以上のみ（発火8本）でも 救った +5.3R / 切った **−16.2R** ＝ −10.9R。
  🔑 **機構（決定的）**: **1時間足の安値を割ったトレードは、そのまま持っていたら平均 +2.649R。ランダムに同数選んだ場合は +1.038R（2パーセンタイル）。** ＝ **構造退出は悪いトレードを避けているのではなく、一番伸びるトレードを選んで切っている。** 大きく走る前には深い押しが入り、それが1時間足の安値を割る。**「もう終わりだ」と感じるその瞬間が、統計的には「これから走る」瞬間。**
- ✅🔑 **【正しい対処】痛み = 賭け率 × R。R の分布はいじらず、賭け率を下げる。これはほぼ無料。**
  btc15m_A（日足×0.75込み）を賭け率だけ変えて回すと: **0.25%→CAGR 8.3%/maxDD 2.2%/CAGR-DD 3.81 · 0.50%→17.1%/4.3%/3.96 · 1.00%→36.1%/8.5%/4.25 · 2.00%→79.9%/16.3%/4.89**。
  **賭け率を半分にする代償は CAGR/DD が 4.25→3.96（−0.29）だけ。出口をいじる代償は同DDで CAGR −10〜16pt。**
  **「RR4.5 を 0.5% で持つ」ほうが「RR1.5 を 1% で持つ」より、感情的に楽で、しかも儲かる**（損切り0.5%・8連敗で−3.9% vs 損切り1%かつエッジ半減）。
  🔑 **一般則: 「怖いから小さく張る」はほぼ無料。「怖いから早く降りる」は一番高い買い物。**
  レジーム側は既に機械化済み（日足 < 150日SMA → サイズ ×0.75）。**相場が戻れば自動で解除される** — ルールがユーザーの代わりにレジームを判断する。

### 2.3 【出口法則の最も深い確認】良い検出器を持っていても、出口には使えない（2026-07-13, btc15m_A）
ユーザーの提案: **「1時間レベルでレンジのような動きになったら利確」**。時間軸そのものも未計測だったので併せて測った。

- ❌🔑 **時間ストップは機構的に不可能**（`experiments/A_hold_time.py`）。**勝ちの保有 14.2時間 vs 負けの保有 4.2時間 ＝ 負けは速く死ぬ。**
  **「h 時間たってもまだ生きている」トレードは、時間が経つほど良くなる**: 最終R（平均）が 全部 +1.085 → 2h生存 +1.580 → 6h +1.975 → 12h +2.251 → **24h +2.332** → 48h +2.471。利確到達率も 35% → 44 → 49 → 53 → **56%**。**構造法則9が「保有中」の領域でも成立**（法則9は元々「波の年齢」で測ったものだったので、これは独立した確認）。
- 🟢🔑 **【本物の検出器】「1時間レベルのレンジ」＝ 直近K時間の値幅 ÷ 損切り幅。ランダム退出nullを通る（4パーセンタイル）**。
  6時間生存の130本を直近6hの値幅で4分割: **Q1（最も狭い、0.90×損切り幅）は 最終R +0.782・利確到達 24%** vs **Q4（3.46×）は +3.463・利確到達 79%**。
  **ランダム退出null: レンジ判定で選ばれた本の「持っていたら」meanR = +0.877 vs ランダム同数 +1.989（中央値）→ 4パーセンタイル。** ＝ **本当に伸びない本を正確に選べている**（朝の「1H安値割れ」は逆に**一番伸びる本**を選んでいた＝2パーセンタイル・上位側）。
- ❌🔑🔥 **【それでも降りると損をする — 出口法則の核心】** 同DD(8.48%)で CAGR: 6h・1.0×退出 **−3.8pt** / 6h・1.5× **−10.1pt** / 12h・1.2× **−3.4pt**。**全12セルが負け。**
  **反実仮想**: 6h・1.0×（発火24本）: 救った +12.1R / **切った −34.0R** ＝ **差引 −21.9R**。
  🔑 **機構（これが答え）**: **「伸びないトレード」は6時間動いていないのだから、定義上 0R 付近にいる。降りると 0R、持つと +0.88R。**
  **正しく見抜いても、降りると損をする。** 損切りが既に −1R で天井を打っているので「ダメなやつを避ける」価値は高々1R。ところが**そのダメなやつでさえ 24% は 4.5R まで走り、0.24×4.5 = +1.08R がすべてを支えている**。
  ∴ **良い in-hold 検出器を持っていても、出口には使えない。** 出口が価値を持つのは「**退出価格 > その時点の期待値**」のときだけで、レンジ判定はその定義上その条件を満たさない（動いていないから退出価格が低い）。
  🟡 **心理的な非常口としてだけ許容できる設定**: **6h・値幅 < 0.8×損切り幅**（発火 年1.3回・代償 CAGR **−0.6pt**）。「完全に死んだトレードから逃げる」規則が心理的に必要なら、これが最も安い。ただし本命は賭け率（§2.2）。

### exit-law-general

- ❌🔑🔥 **【出口法則の核心・これで打ち止め】良い in-hold 検出器を持っていても、出口には使えない**（`experiments/A_hold_time.py` `A_range_exit.py`, 2026-07-13。ユーザーの提案「1時間レベルでレンジのような動きになったら利確」）。
  ✅ **「1Hレンジ」検出器は本物**（直近K時間の値幅 ÷ 損切り幅）: 6h生存130本の四分位で **最も狭い群は 最終R +0.782・利確到達 24%** vs **最も広い群 +3.463・79%**。**ランダム退出nullを 4パーセンタイルで通過**（選ばれた本の「持っていたら」meanR +0.877 vs ランダム +1.989）＝ **本当に伸びない本を正確に選べている**。
  ❌ **それでも降りると損をする**: 同DDで CAGR **−0.6 〜 −10.1pt（全12セル負け）**。反実仮想 6h・1.0×: 救った +12.1R / **切った −34.0R** ＝ 差引 −21.9R。
  🔑 **機構（これが最終的な答え）**: **「伸びない」トレードは動いていないのだから定義上 0R 付近にいる。降りると 0R、持つと +0.88R。損切りが −1R で天井を打っているので「ダメを避ける」価値は高々1R。ところがそのダメなやつでさえ 24% は 4.5R まで走り、0.24×4.5=+1.08R がすべてを支えている。**
  ∴ **出口が価値を持つのは「退出価格 > その時点の期待値」のときだけ。** レンジ判定は定義上その条件を満たさない。**検出器の質の問題ではない。**
  ❌ **時間ストップも機構的に不可能**: 勝ちの保有 14.2h vs 負け 4.2h。**「h時間たっても生きている」本は時間とともに良くなる**（最終R 全部+1.085 → 6h生存+1.975 → 24h生存 **+2.332**、利確到達率 35%→**56%**）。構造法則9が「保有中」領域でも成立。
  🟡 **心理的な非常口としてだけ許容**: 6h・値幅 < 0.8×損切り幅（年1.3回・代償 CAGR −0.6pt）。

### fear-bet-size-not-exit

- ❌🔑🔥 **【最重要の一般則】「持ち続けられない」への対処は、出口ではなく賭け率。「怖いから小さく張る」はほぼ無料、「怖いから早く降りる」は一番高い買い物**（`experiments/h1_swinglow_exit.py`, 2026-07-13。ユーザー申告「戦争中で持ち続けられる相場ではない。1時間レベルのレジサポ転換で利確しようかと」）。
  **「1時間足の直近の押し目安値を割ったら利確」＝ 同じ maxDD(8.48%) で CAGR が 10〜16ポイント落ちる。12セル全滅**（k=2/3/5本 × 発火条件4通り）。最も保守的な「+2R以上でのみ利確」（発火8本）でも **−10.7pt**。
  **反実仮想: どの設定でも切った額が救った額の3倍以上**（k=3・常時: 救った +26.8R / 切った **−73.9R**）。
  🔑 **機構（決定的）**: **1時間足の安値を割ったトレードは、そのまま持てば平均 +2.649R。ランダムに同数選ぶと +1.038R（2パーセンタイル）。＝ 構造退出は「悪いトレードを避ける」のではなく「一番伸びるトレードを選んで切る」。** 大きく走る前には深い押しが入り、それが1時間足の安値を割る。**「もう終わりだ」と感じる瞬間が、統計的には「これから走る」瞬間。**
  ✅ **正しい対処 = 賭け率を下げる**（R の分布は不変）。btc15m_A（日足×0.75込み）: **0.50%→CAGR 17.1%/maxDD 4.3%/CAGR-DD 3.96** vs **1.00%→36.1%/8.5%/4.25**。**半分にする代償は CAGR/DD −0.29 だけ。** 「RR4.5 を 0.5% で持つ」ほうが「RR1.5 を 1% で持つ」より楽で儲かる。詳細 `docs/findings/x_exits.md` §2.2。
