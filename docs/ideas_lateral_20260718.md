# 水平思考の新アイデア10案（2026-07-18、モードA発散）

**趣旨:** レジームゲートの未使用の自由度＋他ドメインからの知識輸入＋価格・時間以外の観測データ、の3軸で発散。
各案は proposals.md と同じ4行形式＋「反証ファーストステップ」（＝その案の一番クリティカルな前提を最初に殺す1スクリプト）＋台帳上の死んだ親戚を付記。
**採用/棄却の判断はユーザー。** 採用された案は docs/proposals.md に転記してモードBへ。

前提の再確認（台帳から）:
- 生きている軸＝サイズ写像（E1スタックPASS・直交成分の枠はまだ食える）・WHENの粒度・新入口族・執行。
- 死んでいる軸＝出口いじり・検出器の追加・動的配分・Binance建玉系（OI/LS/テイカー）・Fear&Greed・月次季節性・FVG埋め/反発。
- 規約: 外部データ(UTC)は必ずブローカー時刻へ変換（tz規約）。層別はランダムラベルの帰無帯と比較。サイズ案は「重み固定」版を併記。
- 外部データ: 無料Webデータは取得可（2026-07-18 ユーザー承認）＝案2/5/8 も実行可能。

**検証の共通順序（2026-07-18 ユーザー指摘で確定: レッグとの組み合わせは最終、最初は単体）:**
1. **単体の情報測定** — レッグと無関係に、変数の分位ごとの「その後の方向」と「その後に動く量」を**分けて**測る。
   方向≈0 でも即死ではない（前例: 時間帯は動く量を3倍差で予言し方向は微小＝サイズ変数の資格は残る）。両方平坦なら完全に死。
2. **冗長性チェック** — 既存ゲート/既存サイズ成分（KAMA・ATR・HH4H）との一致率・相関。9割一致なら縮退で死。
3. **レッグ内の層別** — ここで初めてトレード列に載せる（層別 meanR・ランダムラベル帰無帯）。
4. **単独運用のサイズ写像** — 同DDそろえの CAGR・巡回ブロック・ブートストラップ（E1と同じ物差し）。
5. **ブック裁定は最後** — 重み固定版を併記。
各案の「反証ファーストステップ」は、この順序の中で**その案に固有の最クリティカルな関門**を指す（案3/7/8 のように冗長性が先に立つ案もある）。

## 案1. ブレイク足の出来高（tick_volume）→サイズ写像　【手元データ・未使用の次元】

- **機構:** 全CSVに `tick_volume` 列があり、**一度も使われていない**。ブレイク足の出来高（同時刻帯の中央値比で正規化）は「本物の参加者が居るか」の観測。法則9系「入口の強さはサイズの変数」— 出来高はまだ試していない強さの物差し。
- **残存理由:** FXやCFDのティック出来高はブローカー固有で、論文や市販ツールが扱わない領域。しかも btc15m_L には E1 で実証済みのサイズスタック（受け皿）が既にある。
- **反証ファーストステップ（最クリティカル＝出来高に情報が有るか）:** btc15m_L の全トレードに「ブレイク確定足の出来高 ÷ 過去20日の同時刻帯中央値」を付け、四分位で層別 meanR。単調性が無ければ即死。あればランダムラベル帰無帯と、既存成分（HH4H階段・ATR）との相関で直交性を確認。1スクリプト。
- **予想される死に方:** ティック数はボラの言い換え（ATRと相関0.8超）で、HH4H階段と縮退＝限界利得ゼロ。
- **親戚:** E1スタック（PASS・この枠の増設先例）・ICTラベル（btc単品はnull帯内）。

## 案2. 経済カレンダー近接ラベル→サイズ写像　【台帳自身が残した生きたリード】

- **機構:** 台帳 s15 に「予測可能な"トレンド日"層は価格でなく経済カレンダー」という測定済みリードが放置されている。指標発表（FOMC/CPI/NFP等）は「動くこと」だけが予定されている＝ボラの予約。方向でなく**サイズ**に使う（発表±N時間内の約定は玉を増やす/減らす、両向きを掃引）。
- **残存理由:** 世間のニュース系は方向当て（死屍累々）。「検証済みレッグのサイズ変数として使う」形は、動くレッグと審判を両方持っていないと試せない。
- **反証ファーストステップ（最クリティカル＝層別に実体が有るか）:** 過去の主要指標日時（無料入手）をUTC→ブローカー時刻に変換（tz規約）し、gold15m / btc15m_L の各トレードを「発表±4h/±24h内」でラベル→層別 meanR と MFE中央値。差が出なければ即死。
- **予想される死に方:** gold15m「米指標窓が最良時間帯」（既知）の再発見どまりで、サイズに乗せる限界利得がゼロ。
- **親戚:** セッションスキップ禁止（s08・捨てる窓が黒字）・月次季節性（ボラには実在、損益に移らず）・時間帯の方向情報≈0。

## 案3. 横断ブレッドス・ゲート　【疫学の輸入: トレンドは資産間に感染する】

- **機構:** 感染症の流行判定は「1人の重症度」でなく「何人が罹っているか」。手元の全銘柄（gold/銀/油/指数/FX/BTC）の KAMA上向き比率＝ブレッドスは、単銘柄ゲートと別次元のレジーム観測。エッジの横展開は全滅したが、**エッジの無い銘柄もゲートの情報源にはなりうる**（別の問い）。
- **残存理由:** 株式内部のブレッドスは有名だが、「CFD混成バスケットの上向き比率でBTCレッグを条件付ける」データ構成は誰も持っていない。
- **反証ファーストステップ（最クリティカル＝自銘柄ゲートへの直交性）:** ブレッドス（N銘柄の日足KAMA↑比率）と btc15m_L の4hKAMAゲートの一致率をまず測る。一致率9割超なら冗長で即死。不一致セルの層別 meanR に差があって初めて続行。
- **予想される死に方:** ブレッドスの実体がドル因子＋金銀相関の2変数に縮退（法則2の再発見）。
- **親戚:** レシピ横展開・全滅（s02）・equity-curveメタゲート死（s03）。

## 案4. シグナル発生の群発度（地震学の輸入: 余震・自己励起）

- **機構:** 観測データを価格でも暦でもなく「自レッグのシグナル発生列」にする。地震は群発する＝ブレイクシグナルが密集する期間はレジームが濃い可能性。直近K日の同レッグ・シグナル数→サイズ写像。
- **残存理由:** 戦略固有のシグナル列は自分の検出器の副産物で、世界に存在しないデータ＝定義上未採掘。
- **反証ファーストステップ（最クリティカル＝密度に予測力が有るか）:** btc15m_L の各トレードに「過去14日のシグナル発生数」を付けて四分位層別 meanR。発生時刻シャッフルの帰無と比較。1スクリプト。
- **予想される死に方:** 密度はゲートON期間の言い換え（KAMA上向きなら発生が増える）で縮退。または「走った直後に絞る」撤回と同型のコイン投げ。
- **親戚:** 「BTCが走った直後は絞る」撤回（s07）・D3レジーム年齢死。

## 案5. 資金調達率（funding rate）→ btc15m_S のWHEN　【価格でなく金利の観測】

- **機構:** 無期限先物の資金調達率はロング混雑の直接観測（建玉「量」でなく混雑の「価格」）。混雑ロングの投げがショートの燃料＝ btc15m_S（年12本・弱気年の中和役）に、価格由来でない文脈を供給して濃縮する。
- **残存理由:** 資金調達率は裁定業者がベーシス取りで食う領域で、「15分ブレイク×数日保有の方向トレードの条件付け」という粒度は隙間。
- **反証ファーストステップ（最クリティカル＝日足KAMA↓ゲートへの直交性）:** Binance の資金調達率履歴（無料・8時間毎）をtz変換して結合し、まずKAMA日足ゲートとの相関を測る。直交でなければ即死。直交なら btc15m_S トレードを分位層別 meanR。
- **予想される死に方:** n≈100 の脚では層別が解像しない。または funding が「下げトレンド」の言い換えで直交せず。
- **親戚:** Binance建玉系（OI/LS/テイカー）死＝ただし「KAMA直交だが無力」の前例あり・フロー退出死（tz規約の出所）。

## 案6. 実測スプレッドのレジーム（微細構造: 執行コストの天気）

- **機構:** スプレッドは価格でなく流動性の観測。発注時スプレッドが分布の悪い側なら見送り/半玉。バックテスト不能だが**前向きに検証可能**な衛生規則で、B5（BTC 5分の生死判定）のロガーと同じ段取りで採れる。
- **残存理由:** ブローカー固有のライブデータは文献に存在しない。
- **反証ファーストステップ（最クリティカル＝分布に構造が有るか）:** ロガー数日分で時間帯別スプレッド分布（中央値・p90）を出し、p90/中央値比が2倍を超える時間帯が実在するかだけ見る。無ければルールが発火せず死。
- **予想される死に方:** エントリーが集まる時間帯とスプレッド悪化時間帯が重ならず、適用機会がほぼゼロ。
- **親戚:** B5（ブロック中・ブリッジ起動＋tickエンドポイント追加が前提）。

## 案7. レジームの貸し借り（DXY で gold15m を条件付ける）

- **機構:** 台帳のゲートは全部「自銘柄」の上位足。goldの実ドライバーはドル/実質金利なので、`vantage_usdx.r` の日足KAMAを第2のソフトゲート（下向きでないとき×0.75等）にすると、gold自身のSMA150より速く/別角度でレジームを言う可能性。
- **残存理由:** 「固定ゲートは銘柄固有」の法則は自銘柄ゲート間の比較から出た。クロス銘柄ゲートは未検証の自由度。
- **反証ファーストステップ（最クリティカル＝自銘柄ゲートへの直交性）:** USDX日足KAMAの向きと gold15m トレードの層別 meanR＋SMA150ゲートとの一致率。一致率9割超なら縮退で即死。
- **予想される死に方:** DXY↓とgold日足↑がほぼ同義で縮退。またはUSDXデータが2015年以降しかなく時代検定が薄い。
- **親戚:** ICTのDXY方向ゲート（残ったのはドル・ベータのみ）・単一ドル因子の疑い（法則2）。

## 案8. オンチェーン低速指標（MVRV等）→ BTC のサイクル文脈

- **機構:** 取引所データでなくブロックチェーン台帳の観測（保有者の含み益比率など）。週足30MAサイクルゲート（btc_pull用）の代替/補強として、価格から独立した低速サイクル位置を供給。
- **残存理由:** 低速（週次）×トレンドレッグのサイズという組み合わせは高頻度勢の対象外。
- **反証ファーストステップ（最クリティカル＝入手性と直交性）:** 無料で信頼できる履歴（coinmetrics community等）が取れるかをまず確認 → 週足30MAゲートとの一致率。高ければ即死。
- **予想される死に方:** サイクル位置系は「符号が逆」の前科（s03）どおり死。もしくはデータの改訂・欠測で先読み禁止が守れない。
- **親戚:** Fear&Greed死・サイクル位置ゲート符号逆・Binance建玉系死。

## 案9. 上位足の中の位相（15分シグナルが4H足のどこで出たか）

- **機構:** 元の問い「4H/1Hから15分を条件付けられるか」の機械化。ゲートは4H足の**確定値**しか見ていない。同じブレイクでも「4H足の始まり直後」と「4H足が既に大陽線の終わり際」では文脈が違うはず。足内の経過本数(0〜15)と足内位置で層別→サイズ。
- **残存理由:** 追加データ不要・完全に手元で閉じる未使用の自由度。多くの検証は上位足を「確定後」しか使わない。
- **反証ファーストステップ（最クリティカル＝位相で層別が有るか）:** btc15m_L の各トレードに位相2変数を付けて層別 meanR。平坦なら即死。1スクリプト・最安。
- **予想される死に方:** 時間帯の方向情報≈0（s01）の親戚として平坦。
- **親戚:** 時間帯そのものの方向情報「ゼロではないが食えない」・ゲートの窓×サンプリング則。

## 案10. 相関レジーム（生態学の輸入: 多様性が消える瞬間）→ブック総リスク

- **機構:** ブックの価値の源泉はレッグ間の無相関（法則6/10）。危機時は全資産の相関が1に寄る＝分散が消える。手元銘柄の実現相関平均が跳ねた時だけ**ブック総リスクのスカラー**を絞る（レッグ配分はいじらない＝動的配分の死因を回避）。
- **残存理由:** 死んだのは「レッグ間の配分」の動的化。「総リスク1本のスカラー」を外部観測で絞る形は未検証。
- **反証ファーストステップ（最クリティカル＝タイミングの先行性）:** 30日実現相関の平均系列を作り、ブックの上位5DD局面と重ねる。相関スパイクがDD**開始後**にしか立たないなら価値ゼロで即死。
- **予想される死に方:** 相関は事後にしか上がらず、絞る頃にはDDが終わっている（動的配分全滅の仲間入り）。
- **親戚:** equity-gate死・動的アロケ死・静的inv-vol最強（s07）。

### 案2の再現失敗（プロセス上の重要事項）
初回は Wayback Machine のスナップショットが取れて FOMC72/CPI109/NFP109=290件で、gold15m の発表±4h窓に「次4hの動く量が中央値6割増・帯の外」という強い信号が出た。
**だが再実行では Wayback が全て HTTP 500 で落ち、FOMCライブ48件のみに縮小＋良い方のCSVを上書き**してしまった。
= **信号自体は実世界的にもっともらしい（gold は FOMC/CPI/NFP で大きく動くのは既知）が、スクレイピングが非決定的で今の形では数字を保証できない**。
追う場合の最初の一手＝**決定的な指標カレンダー源の整備**（Wayback依存をやめる。日付は静的事実なので固定リスト化が可能）。それまで案2は「未検証・保留」。

### 方針転換（2026-07-18 ユーザー指定）: サイズ写像でなく「取る/見送るフィルター」で出す
ユーザーは 0.01 ロット固定でしか張らない＝サイズ変数は実装価値ゼロ。変数は**シグナル強度のしきい値（取る/見送る）**として出す。
期待値の評価は内部で R・ロット換算も併記するが、出力する signal はあくまで強度ベースの go/no-go。
∴ 前掲の STEP4「サイズ写像」は全案で破棄し、STEP3（レッグを固定ベットでフィルター）が最終測定になる。
固定ロットでフィルターが助かるのは「見送った部分集合が EV 負（or 十分低い）」の時だけ＝高いハードル（フィルターは濃縮であって創造でない・法則3/4/7）。

### STEP3 へ進める最有力＝案1（出来高）をフィルター化
方向ゼロ・量を強く単調に予言＝「出来高の大きいブレイクほど、その後よく動く」。遠い固定目標（RR4.5）のレッグでは、
**よく動く＝目標到達率が上がる**可能性があり、固定ベットでも meanR/totR を押し上げうる（サイズ不要でフィルターとして効く筋）。
仕様（凍結前の骨子・測定のみ・ブック不変）:
1. 対象: btc15m_L と gold15m の全シグナル・トレード。変数 X = ブレイク確定足の出来高比（STEP1と同じ因果的定義）。
2. フィルター: X ≥ しきい値のトレードだけ残す（しきい値を分位で掃引）。ベースライン（全トレード）と並べて
   meanR・PF・win%・totR・maxDD(トレード解像度)＋**0.01ロットでの1トレード平均ドル損益**を報告。
3. 見送った部分集合の meanR/PF を明示（固定ロットでの機構＝切った側がEV負か）。
4. null: ランダム除去（同数を1000回）で totR/DD のパーセンタイル＋巡回ブロック・ブートストラップ1/3/6/12mo（本物は長ブロックで勝率上昇）。
5. 冗長性: X と既存の入口フィルター（PDH・ext-cap・4hKAMAゲート）との重複率。
予想される死に方: (a) 出来高比が ATR/ボラの言い換えで、既存の損切り幅（ATR基準）と縮退し切った側もEV正＝濃縮せず、
(b) 高出来高ブレイク＝ニュース・スパイクで逆に続かず、切った側のほうがEV高い（案1の"量"は"質"でない）。
