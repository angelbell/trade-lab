# 仕様カード3 — 「ゲート指標の傾きの急さ」は他レッグでも強度になるか（横断プリミティブ検定）

## 背景（カード2の結果）
btc15m_L で kama_slope（4H KAMA(14) の1本あたり傾きの大きさ）が弱いが本物の強度勾配を示した:
Spearman +0.088、ブロックbootstrap CI 下限が1〜12か月とも0超、時代ベータ隔離も通過（急KAMAの優位は
2020-2021強気で負・2022弱気で最大＝方向ベータでなくトレンド加速を拾う）。対して daily_trend（水準乖離）と
mtf_agree（合流度）は失格。∴ 効いたのは「傾き＝加速」で、しかも btc15m_L 自身のゲート指標。

## 問い（横断プリミティブか）
「各レッグが、自分のトレンドゲート指標の傾きの急さで per-trade EV(meanR/PF) を並べ替えるか」。
1レッグだけなら過剰適合。gold15m と btc15m_S でも同じ形が出れば横断の強度プリミティブ（構造法則1の逆用）。

## 対象レッグ2本（book.py get_book_legs と厳密一致で再構築・照合ゲート必須）
1. gold15m（L94-98）: 強度候補 = 日足 SMA150 の傾きの急さ。
   - slope = (SMA150_daily[i] - SMA150_daily[i-1]) / SMA150_daily[i-1]、前日までの確定日足。
   - gold15m のゲートは daily_sma=150（上向き条件）。その傾きの大きさが強度。
2. btc15m_S（L107-112, ショート・鏡像）: 強度候補 = 日足 KAMA(14) の傾きの急さ（下向き＝負に急）。
   - gate は日足KAMA↓。強度 = |傾き|（下向きに急なほど強い）。符号の扱いを明記（下降加速で強度大）。
   - ショートは反転系列 inv で建てるので、R はショートPnL。強度と R の向きを取り違えないこと。

## 照合ゲート（各レッグ・カード1/2と同型・必須）
- 自作 netR が book.get_book_legs()[leg] と時刻・値で配列一致（gold15m は 0.3/risk コスト、btc15m_S は
  15/risk コスト＋PDLハードmask を book と同じに）。一致後のみ層別。entries↔確定足 i 対応も確立。
- gold15m/btc15m_S の entries 直呼び再構築は strength_btc15mL.py の関数を流用（pullback_frac 等の引数だけ差し替え）。
  btc15m_S は inv 系列・PDLマスク・rr_short=4.5 を book と一致させる。

## 測り方（カード2と同一）
- 各レッグ・各候補: 5分位で n/win%/PF/meanR/totR、単調性、Spearman(変数vs実現R)、
  巡回ブロック・ブートストラップ(1/3/6/12か月,各1000回)95%CI、ランダム除去null percentile。
- 時代ベータ隔離: 年別 Q5-Q1 meanR（probe_kama_slope_era.py と同型）。符号が単一年に依存しないか。
- 報告は PF・N・meanR 併記。金額換算不要。scratchpad/strength_gateslope_generalize.py。

## 判定
- gold15m・btc15m_S の少なくとも一方で Spearman 正・ブロックCI 0超・時代分散 が btc15m_L と同傾向
  → 横断の強度プリミティブとして採用候補（3レッグの傾き強度スケールを README に併記）。
- 両方フラット/反転 → kama_slope は btc15m_L 固有（法則1）＝1レッグの裁量サイジング補助にとどまる。

## 死に方（予想）
- gold15m: 日足SMA150 は KAMA より遅い（150本平均）ので傾きが鈍く、分位が潰れて識別力が出ない疑い。
  その場合は「同じ加速でも指標の速さで効きが変わる」＝法則1（TF×method固有）の実例として記録。
- btc15m_S: ショートはドリフト逆行（法則11）で母数が薄く（年12本級）、分位あたりnが小さすぎて
  ブロックbootstrapが常に0をまたぐ疑い。n不足なら「測定不能」と明記（無理に有意化しない）。
