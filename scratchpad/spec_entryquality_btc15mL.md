# 仕様カード5 — btc15m_L「エントリーの質＝実効リスクの小ささ」を強度として測る

## 背景・狙い
ユーザーの口座制約＝1トレードのstop$（0.01ロット×stop距離）。明言「損切りタイトで完璧なエントリーなら大きく張れる」。
∴ kama_slope(トレンド加速, 採用済み弱シグナル)とは別軸で「**エントリー質＝実効リスクの小ささ**」が per-trade EV を
並べ替えるかを検定。並べ替えれば「タイト/完璧な入口＝大きく張る」に数値の根拠。単体測定（kama_slope合成は後段）。

## 対象・土台
btc15m_L 1本。`scratchpad/strength_btc15mL.py`（entries直呼び再構築＋トレード↔確定足i対応、照合ゲート3本
=book.get_book_legs()['btc15m_L']と配列一致 PASS済み）を import して土台にする（再発明禁止）。確定足i配列取得済み。
R は素netR = tL.R - 15/tL.risk（PDH重みは別軸なので外す）。

## 強度候補（3つ・単体で・すべて確定足iで評価・no-lookahead）
1. stop_atr = risk / ATR(14)[i]（risk=tL.risk=stop距離、ATRは15m足の確定足まで）。損切りがボラ対比でタイトか。
   ※risk_frac(=risk/price)はカード1で死んだが、これはATR正規化＝別変数。「タイトほど良い」なら低stop_atrで
   meanR高。仮説の向きは決めず測る。
2. support_dist = (fill価格 e_px − 直近スイング安値) / ATR(14)[i]。スイング安値=確定足iまでの過去K本の最安値
   (K=48目安, rolling min, 先読みなし)。小さいほど「支えの直上で入った＝完璧な入口・stopが構造に守られる」。
3. atr_pctile = ATR(14)[i] の過去分位（trailing 500本目安の percentile）。低いほど静かな相場＝1発の$リスク小
   （口座制約に直結）。ボラ水準そのものが EV を並べ替えるか。

## 測り方（カード2-4と同一）
各候補: 5分位で n/win%/PF/meanR/totR、単調性、Spearman(変数vs実現R)、
巡回ブロック・ブートストラップ(1/3/6/12か月,各1000回)95%CI、ランダム除去null percentile、時代ベータ隔離(年別Q5-Q1)。
報告はPF・N・meanR併記、金額換算不要。scratchpad/strength_entryquality_btc15mL.py。実行 .venv/bin/python。

## 判定
- どれかが Spearman有意・ブロックCI下限0超・時代分散 → 「エントリー質」が第2の強度軸として採用候補
  （次段で kama_slope との2次元＝直交すれば合成、被れば冗長を測る）。
- 全滅 → エントリー質は btc15m_L のEVを並べ替えない＝「完璧な入口で大きく張る」は直感だが数値の裏付け無し
  （その場合ユーザーの制約は"サイズを上げる根拠"でなく"stop$の上限"としてのみ効く、と明記）。

## 死に方（予想）
- stop_atr: RR固定(4.5)なのでタイトstopは目標も近く win%上がるが、それはRR力学であってEVでない疑い
  （risk_fracが死んだのと同型。ATP正規化で復活するか否かが論点）。
- support_dist: 「支えの直上」は良い入口に見えて、実は最も走るトレードは支えを一旦割ってから走る
  （構造トレールが負ける機構＝法則4/9の入口版）＝逆単調 or フラットの疑い。
- atr_pctile: 静かな相場のブレイクは伸びない/ダマシ多い可能性（squeeze-breakout がトレンド資産で死んだ機構）
  ＝高ボラのほうが良い＝逆向きの疑い。どちらでも単調性が出れば強度軸。

## 追加パート B — stop$ 可否フィルタ（ユーザーの実制約に直結）
背景: ユーザー制約＝1トレードのstop$（0.01ロット×stop距離）。gold4H は stop$ が口座に対し遠すぎて
「そもそもエントリー対象にならない」（可否の話・サイジングでない）。btc15m_L でも同じ足切りが要るか測る。
- BTC 0.01ロットの 1ポイント($)価値を確認して、各トレードの stop$ = tL.risk × (0.01ロットの$/point) を出す。
  ※契約仕様が不明なら、まず tL.risk（価格単位のstop距離）そのものの分布で代用し、$換算は前提を明記して併記。
- 報告: stop距離($換算)の分布（中央値・分位・最大）、年別の中央値。
- 可否フィルタの影響: 「stop$ が上限X以下のトレードだけ取る」と、残る頻度(n/yr)と meanR/PF/totR/CAGR-DD が
  どう変わるか。Xを複数（分布のp50/p70/p90 目安）掃引。**遠stopトレードを外すとEVが上がるのか下がるのか**を明示
  （外して悪化なら「制約は"取れないから諦める"であってフィルタの利得ではない」と正直に書く）。
- これはサイジングでなく可否（take/skip）の話。強度スケールとは独立に報告する。
