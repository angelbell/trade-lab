# 仕様カード2 — btc15m_L を「レジーム強度」で層別し、EV単調性を測る

## 背景（カード1の結果）
内部形状3候補（base_bars=wave長 / risk_frac=損切り幅 / vol_z=確定足出来高）は全て強度勾配ゼロ
（Spearman ±0.03、単調性NO、ブロックbootstrap CIが0をまたぐ）。病巣＝いずれも「そのブレイク自体の幾何」で、
4H KAMA↑ゲートで既に濾された後のトレードの中では EV を並べ替えない（法則9・vol_zのSTEP1所見と一致）。
∴ 次は「トレンド文脈の強度」＝ゲートの満たし方の強さ を測る（法則3: WHEN が最大レバー）。

## 目的
btc15m_L の発火トレードを **レジーム強度スコア** で5分位に切り、meanR/PF が強度に沿って単調に上がるか。
上がれば裁量サイジング（強いトレンド→大きく張る）に根拠。フラット/反転なら強度失格。自動サイズ写像は組まない。

## 対象・照合ゲート（カード1と同一・再掲）
btc15m_L を research/book.py get_book_legs() L100-105 と厳密一致で再構築。
scratchpad/strength_btc15mL.py の entries 直呼び再構築（照合ゲート3本 PASS 済み）をそのまま流用してよい
（車輪の再発明禁止＝そのスクリプトを import か複製して土台にする）。トレード↔確定足 i の対応は確立済み。

## レジーム強度候補（3つ・単体で・すべて確定足 i で評価・no-lookahead）
1. kama_slope: 4H KAMA(14) の傾き。ゲートは KAMA↑の二値。強度 = (KAMA[i] - KAMA[i-1]) を
   価格 or ATR で正規化した大きさ。HTF(240min)は shift/confirm-later で決定足までの確定値のみ使う
   （既存 gate_kama_tf="240min" の計算経路と同じ足付けにすること）。
2. daily_trend: 価格の日足SMA150 からの上方乖離 = (close_daily[i] - SMA150_daily[i]) / SMA150_daily[i]。
   日足は前日までの確定値（当日足は使わない）。トレンド確信度の代理。
3. mtf_agree: 合流度 0-3 の整数 = 次の3条件の真の数を確定足 i で数える:
   (a) 4H KAMA(14) 上向き（この足は必ず真＝ゲート条件。強度の足しにはならないので、
       代わりに「4H KAMA の傾きが中央値より急」を条件(a)に置き換えてよい。実装者判断で明記）、
   (b) 日足 SMA150 上向き（傾き>0）、(c) 週足トレンド（終値 > 30週MA、or 週足終値上昇）。
   ※合流度は順序尺度なので5分位でなく 0/1/2/3 のバケットで meanR/PF/n を出す。

## 測り方（カード1と同一）
- 各候補: 5分位（mtf_agree は 0-3 バケット）で n/win%/PF/meanR/totR。分位1→5(or 0→3)で単調に上がるか明示。
- Spearman（変数 vs 実現R）＋巡回ブロック・ブートストラップ(1/3/6/12か月,各1000回)95%CI＋ランダム除去null percentile。
- 報告は PF・N・meanR 併記。金額換算は不要。

## 死に方（予想）
- kama_slope: 急なKAMAは「強いトレンド」だが、既にブレイク後の押し目指値で入っているので、
  急な時ほど押しが浅く約定しにくい/裾を取り逃す可能性 → meanR フラットの疑い。
- daily_trend: 乖離が大きい = 伸びきった後 = mean-reversion で meanR 低下の疑い（逆単調もありうる、要確認）。
- mtf_agree: 合流が多いほど良く見えるが、合流の大半が「上昇相場」に集中＝時代ベータの層別に化ける疑い
  （年別でON率が偏らないか、ブロック伸長で勾配が残るかで判別）。
