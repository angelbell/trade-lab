# 仕様カード11 — btc15m_A に合成強度旗を移植＋既存ロット調整を強度に変換

## 目的（2つ・ユーザー指示 2026-07-19）
1. **案1**: 単体最良レッグ btc15m_A に、btc15m_L で確立した合成強度旗（kama_slope＋stop_atr＋atr_pctile の
   等重みランク平均・トップ20%）を移植。A は L と同じ検出器・執行・銘柄・TF（母集団を新値圏 entry>PDH に絞っただけ）
   なので、他レッグ移植が全滅した「型違い」問題が無い＝転移の見込みが最も高い。
2. **ロット調整→強度変換**: A の単体仕様に含まれるサイズ成分（日足レジーム↓のとき×0.75、台帳 s02）を、
   ロット倍率でなく**signal強度の段**として出し直す（[[feedback-fixed-lot-no-sizing]]: 出力は強度・
   実運用はユーザーが裁量で強→大/弱→小に張る）。**サイズ倍率の推奨は一切出力しない。**

## 対象の再構築（照合必須）
btc15m_A = README「現行の btc15m_L から『前日高値の中でのブレイク』を落とすだけ」＝ **kama4h ゲート版・
entry>PDH ハード選別・PDHソフトサイズ無し**。台帳の基準数字: n238 / win30% / meanR+0.941 / PF2.27 /
IS/OOS +0.92/+0.96（`docs/findings/s13_pullback_limit.md` の kama4h(C1攻め)版）。
- 土台: `scratchpad/strength_btc15mL.py` の build/rebuild_entries/match_entries_to_trades（照合PASS済み）＋
  `src/engine/size.py` の PDH 計算（pdh_soft の PDH 系列を流用し、e_px>PDH ハード選別に使う）。
- **照合ゲート**: 再構築した A が台帳基準（n≈238・meanR≈+0.94・PF≈2.27）と一致することを最初に確認。
  一致しなければ以降の数字は出さず不一致点を報告。R はネットコスト15/risk込み。

## パートA — 合成強度旗の移植
- A のトレードに、L と同一定義の3軸（kama_slope[確定足i]・stop_atr=risk/ATR14[i]・atr_pctile=ATR trailing500分位[i]）
  を付け、**A の母集団内でランク**を取り直して合成スコア＝等重みランク平均。
- トップ20% vs 残り: n/win%/PF/meanR、ギャップ、巡回ブロックブートストラップ(1/3/6/12mo,3000回)95%CI、
  年別 top−rest（時代隔離）。5分位表も参考で。
- **冗長性チェック**: A は既に新値圏で濃縮済み。旗の3軸それぞれについて A 母集団内の単体 Spearman を出し、
  L で見えた勾配が新値圏選別に食われていないか（特に stop_atr）を明示。

## パートB — 日足レジームのロット調整→強度変換
- A のトレードを日足レジーム（×0.75の条件と同一定義: 日足下降トレンド。台帳の定義を確認して使う。
  例: 日足KAMA(14)↓ or 台帳/README記載の定義。**定義が曖昧なら報告して確認を求める**、推測で進めない）で層別し、
  日足↑ / 日足↓ の n/win%/PF/meanR を出す。
- 出力は「日足↓＝弱段（meanR がこれだけ低い）」という**強度の段**として表現。×0.75という倍率は出力しない。

## パートC — 統合強度表（納品物）
合成旗（トップ20%/残り）× 日足レジーム（↑/↓）の 2×2 で n/win%/PF/meanR/年本数。
「A のトレードはこの表で強弱を読む」という**裁量用の一枚**に落とす。セルが薄ければ n を明記し無理に読まない。

## 判定
- パートA: トップ旗ギャップ＋ブロックCI下限0超＋時代分散 → A でも旗が生きる＝最良レッグに強弱部品が付く。
  フラット → 「新値圏濃縮が旗の情報を食っていた」を冗長性チェックで機構まで特定。
- 報告は PF・N・meanR 併記。scratchpad/strength_btc15mA.py。実行 .venv/bin/python。

## 死に方（予想）
- stop_atr: 新値圏ブレイクは構造が大きい側に偏っている＝母集団内で分散が痩せて勾配が消える。
- atr_pctile: 新値圏（走っている相場）は高ボラに偏る＝同上。
- kama_slope: ゲートは同じ kama4h なので L と同様に残る可能性が最も高い。
- 3軸とも痩せたら「A は旗の情報を選別として既に使い切っている」＝それ自体が綺麗な機構の説明。
